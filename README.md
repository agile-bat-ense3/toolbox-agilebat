# Toolbox AGILE-BAT — Brief site web

## Contexte

Projet d'école à Grenoble INP – ENSE3, parcours PARIN 2e année.
Sujet : "Processus agile pour une meilleure prise en compte des performances énergétiques du bâtiment en phase de conception amont".

Livrable : un site web pédagogique pour étudiants en ingénierie et architecture, qui propose une FAQ orientant vers le bon outil énergétique selon la question posée et la phase du projet (Esquisse ou APS).

## Stack technique souhaitée

- HTML + CSS purs, pas de framework
- Pas de build step, pas de dépendances npm
- Hébergement prévu sur GitHub Pages
- Le code doit être lisible et maintenable par d'autres étudiants après nous

## Design

- Sobre, académique, lisible
- Pas de gradient, pas de glassmorphism, pas d'animation gratuite
- Typographie système ou Inter
- Palette neutre (gris/noir/blanc) + une couleur d'accent unique
- Inspiration : documentation technique propre type MDN ou docs Stripe, pas startup tech
